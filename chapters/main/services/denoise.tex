\section{Thiết kế dịch vụ khử nhiễu}

Dịch vụ khử nhiễu được thiết kế theo kiến trúc layered gồm 3 tầng: API,
Service và Model. Tầng API tiếp nhận file WAV qua endpoint
\texttt{POST /denoise} (multipart/form-data, tối đa 50 MB) và cung cấp
endpoint \texttt{GET /health} để giám sát trạng thái. Tầng Service chịu trách
nhiệm tiền xử lý audio (validate sample rate, chuyển stereo sang mono) và hậu
xử lý kết quả. Tầng Model quản lý mô hình DTLN theo Singleton Pattern --- model
được load một lần duy nhất khi khởi động và cache trong bộ nhớ, tránh chi phí
khởi tạo lặp lại cho mỗi request.

Framework được chọn là Flask 2.3+ kết hợp TensorFlow 2.13+, phù hợp với yêu
cầu xử lý đồng bộ của pipeline khử nhiễu vì mỗi request cần chờ model xử lý
xong trước khi trả kết quả.

\subsection{Pipeline xử lý khử nhiễu}

Khi nhận request, dịch vụ đọc file WAV qua thư viện SoundFile và validate
rằng sample rate là 16 kHz --- nếu không, trả về lỗi 400 Bad Request.
File stereo (nếu có) được convert sang mono bằng cách lấy trung bình hai kênh,
sau đó cast sang float32 trong phạm vi $[-1, 1]$. Tín hiệu được thêm batch
dimension và đưa qua \texttt{model.predict()} để khử nhiễu. Đầu ra được clip
vào phạm vi $[-1, 1]$ để tránh clipping khi ghi file, sau đó lưu thành file
WAV tạm thời và trả về client qua HTTP response. File tạm được xóa ngay sau
khi gửi xong để tránh tích tụ trên disk.
